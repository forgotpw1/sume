ActionView::Helpers::RecordTagHelper#content_tag_for

content_tag_for creates an HTML element with id and class parameters that relate to the specified Active Record object. For example: <%= content_tag_for(:tr, @person) do %> <%= @person.first_name %> <%= @person.last_name %> <% end %> would produce the following HTML (assuming @person is an instance of a Person object, with an id value of 123): .... If you require the HTML id attribute to have a prefix, you can specify it: <%= content_tag_for(:tr, @person, :foo) do %> ... produces: ... You can also pass an array of objects which this method will loop through and yield the current object to the supplied block, reducing the need for having to iterate through the object (using each) beforehand. For example (assuming @people is an array of Person objects): <%= content_tag_for(:tr, @people) do |person| %> <%= person.first_name %> <%= person.last_name %> <% end %> produces: ... ... content_tag_for also accepts a hash of options, which will be converted to additional HTML attributes.
